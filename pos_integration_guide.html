【小売業者様向け】POS連携SDK 組み込みガイド

本SDK（pos_connect_sdk.js）は、既存のPOSシステムに「どこでもレジ」を瞬時に連携・導入するための魔法のスクリプトです。
貴社のPOSプログラミングにこのSDKを1行コピーして組み込むだけで、
既存レジの削減、レジ待ち行列の解消、そして商品マスターの自動クラウド同期が即座に実現し、
お客様は自分のスマホで並ばずに決済（モバイルチェックアウト）ができるようになります。

組み込み場所の具体例

パターンA：WebベースのPOSレジ（店舗タブレット等）の場合

貴社の既存POSレジのプログラム内で
本SDKを読み込ませるだけで、
店舗の商品データとクラウドが自動同期され
顧客用スマホレジ（どこでもレジ）が稼働開始します。

// 貴社POSの決済完了処理の例 function onPaymentSuccess(transactionInfo) { // 既存の自社POS処理（レシート印字や自社DBへの保存など） printReceipt(); saveToDatabase(); // ⬇⬇ ここにリテアドのSDKを読み込んで実行！ ⬇⬇ const retailAd = new RetailAdPOS('発行された専用APIキー'); retailAd.trackSale({ transactionId: transactionInfo.id, amount: transactionInfo.totalPrice, items: transactionInfo.itemNames }); }

パターンB：ECサイトや外部決済システム（GMO・SBI等）をご利用の場合

ECサイトや自社アプリに「どこでもレジ」の決済同期を組み込む場合、ユーザーが決済を終えて最後に表示される「完了画面（サンクスページ）」にスクリプトを貼り付けるか、またはバックエンドから以下のAPIへ直接送信してください。

💡 サーバー間API連携（Webhook対応）

GMOペイメントゲートウェイやSBIルミエスなどの決済APIをご利用の場合、決済完了のWebhookを受け取ったタイミングで、REST API経由で直接弊社のサーバーへデータをPOST送信するだけで「どこでもレジ」システムと完全に同期されます。
送信先エンドポイント: https://貴社のAppRunnerドメイン/api/pos/transaction

<!-- 1. SDKを読み込む --> <script src="https://貴社のAppRunnerドメイン/pos_connect_sdk.js"></script> <!-- 2. 売上データを送信する --> <script> const retailAd = new RetailAdPOS('発行された専用APIキー'); retailAd.trackSale({ transactionId: 'TX-98765', amount: 2980, items: ['商品A', '商品B'] }); </script>

実装時の重要な注意点

  1. 二重送信の防止: ユーザーが完了画面（サンクスページ）を「再読み込み（リロード）」した際に、売上が2回カウントされないよう、システム側で制御するか、セッションを使って1度の購入につき1回だけデータが送信されるように設計してください。
  2. 設置ページの限定: 商品一覧ページやカート画面など、購入が確定していないページには絶対に貼らないでください（架空の売上が計測されてしまいます）。
  3. アイテム名の指定: items に渡す商品名は、リテアドの広告主が設定している「キャンペーン対象商品名」と一致するか、商品コード（JANコード等）を送る運用をお勧めします。
