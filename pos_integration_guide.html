【小売業者様向け】POS連携SDK 組み込みガイド

本SDK（pos_connect_sdk.js）は、貴社のPOSシステムやECサイトの「決済・購入が完全に成立したタイミング」にのみ実行されるよう、以下のガイドに従って組み込んでください。

SDK本体のソースコードを確認する

組み込み場所の具体例

パターンA：WebベースのPOSレジ（店舗タブレット等）の場合

レジ画面のプログラム内で、クレジットカード決済や現金決済が「完了」し、完了画面へ切り替わる処理（関数）の中に貼り付けてください。

// 貴社POSの決済完了処理の例 function onPaymentSuccess(transactionInfo) { // 既存の自社POS処理（レシート印字や自社DBへの保存など） printReceipt(); saveToDatabase(); // ⬇⬇ ここにリテアドのSDKを読み込んで実行！ ⬇⬇ const retailAd = new RetailAdPOS('発行された専用APIキー'); retailAd.trackSale({ transactionId: transactionInfo.id, amount: transactionInfo.totalPrice, items: transactionInfo.itemNames }); }

パターンB：ECサイト（オンラインショップ）の場合

ユーザーがクレジットカード決済等を終え、最後に表示される「ご購入ありがとうございました（サンクスページ/完了画面）」のHTMLの <body> タグの下部に以下のスクリプトを貼り付けてください。

<!-- 1. SDKを読み込む --> <script src="https://貴社のAppRunnerドメイン/pos_connect_sdk.js"></script> <!-- 2. 売上データを送信する --> <script> const retailAd = new RetailAdPOS('発行された専用APIキー'); retailAd.trackSale({ transactionId: 'TX-98765', amount: 2980, items: ['商品A', '商品B'] }); </script>

実装時の重要な注意点

  1. 二重送信の防止: ユーザーが完了画面（サンクスページ）を「再読み込み（リロード）」した際に、売上が2回カウントされないよう、システム側で制御するか、セッションを使って1度の購入につき1回だけ発火するように設計してください。
  2. 発火ページの限定: 商品一覧ページやカート画面など、購入が確定していないページには絶対に貼らないでください（架空の売上が計測されてしまいます）。
  3. アイテム名の指定: items に渡す商品名は、リテアドの広告主が設定している「キャンペーン対象商品名」と一致するか、商品コード（JANコード等）を送る運用をお勧めします。
