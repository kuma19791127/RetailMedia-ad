AWS App Runner でアウトバウンド（外部への送信）IP アドレスを固定化するための、具体的なインフラ構築の手順を解説します。

この設定を行うことで、App Runner から GMOあおぞらネット銀行の API にアクセスする際の IP アドレスが **Elastic IP (固定のグローバルIP)** に固定され、GMO側に正しい IP アドレスを申請できるようになります。

---

### 全体構成のイメージ
```mermaid
graph TD
    AppRunner[AWS App Runner] -->|VPCコネクタ| PrivateSubnet[プライベートサブネット]
    PrivateSubnet -->|ルートテーブル 0.0.0.0/0| NATGW[NAT Gateway + Elastic IP]
    NATGW -->|固定IPで通信| Internet((インターネット))
    Internet -->|IP制限で許可| GMO[GMOあおぞらネット銀行 API]
```

---

### 構築の具体的なステップ

AWSの管理コンソールから以下の手順で設定を行います。

#### ステップ 1: VPC（仮想ネットワーク）を作成する
すでにVPCがある場合はそれを利用できますが、クリーンな環境を作るために新規作成することをおすすめします。AWSの「VPCを作成」ウィザードを使うと一瞬で作成できます。

1. AWSコンソールで **VPC** サービスを開きます。
2. **「VPC を作成」** ボタンをクリックします。
3. 設定ウィザードで以下のように選択します：
   * **作成するリソース**: 「VPC など」（関連リソースもまとめて自動作成してくれます）
   * **名前タグ**: `retail-media-vpc`（任意）
   * **IPv4 CIDR ブロック**: `10.0.0.0/16`
   * **可用性ゾーン (AZ) の数**: `2` （冗長性のために2つ必要です）
   * **パブリックサブネットの数**: `2`
   * **プライベートサブネットの数**: `2`
   * **NAT ゲートウェイ**: **「1つの AZ 内」** または **「AZ ごとに 1 つ」** を選択します。
     * ※「1つの AZ 内」にすると、NAT Gateway
<truncated 1362 bytes>
PC (`retail-media-vpc`) を選択します。
   * **サブネット**: 作成した **プライベートサブネット（Private Subnet）を2つ** 選択します。
     * ※必ず「Private」と名のつくサブネットを選択してください。
   * **セキュリティグループ**: デフォルトのセキュリティグループ、またはアウトバウンド（送信）が許可されているセキュリティグループを選択します。
4. **「作成」** をクリックします。

---

#### ステップ 3: App Runner サービスに VPC コネクタを適用する
稼働中の App Runner サービスのネットワーク設定を変更します。

1. App Runnerのサービス一覧から、対象のサービス（`nsg3hyme2k`）を選択します。
2. **「設定」** タブを開き、**「ネットワーク」** セクションの **「編集」** ボタンをクリックします。
3. **「発信トラフィック (送信トラフィック)」** の設定を以下のように変更します：
   * **「カスタム VPC」** を選択します。
   * **VPC コネクタ** に、ステップ2で作成した `retail-media-vpc-connector` を選択します。
4. **「変更を保存」** をクリックします。サービスの再デプロイ（ローリングアップデート）が自動で始まり、数分で完了します。

---

### 3. GMOへ提出する「固定IPアドレス」の確認方法

上記の設定が完了したら、NAT Gatewayに紐づいている **Elastic IP** が、GMOあおぞらネット銀行へアクセスする際の本番IPアドレスになります。

1. AWSコンソールで **VPC** サービスを開きます。
2. 左メニューの **「Elastic IP」** をクリックします。
3. NAT Gatewayに関連付けられている Elastic IP アドレス（例: `54.150.xx.xx` など）がリストに表示されています。
4. **このIPアドレスを、GMOの「API利用申込書」の本番環境IPアドレスとして記入します。**

2. **「VPC を作成」** をクリックします。
3. ウィザードで以下を選択します：
   * **作成するリソース**: 「VPC など」
   * **NAT ゲートウェイ**: **「なし」**（ここがコスト削減のポイントです）
   * その他の設定（パブリックサブネット2つ、プライベートサブネット2つ、可用性ゾーン2つ）はデフォルトのままで **「VPC を作成」** をクリックします。
